Why VLLM Studio?

Existing local AI chat tools are built around llama.cpp or HuggingFace Transformers. vLLM delivers 10x+ higher serving throughput via PagedAttention and continuous batching, but lacks a polished desktop management interface. VLLM Studio fills that gap.

Feature Highlights

⚡

One-Click vLLM Management

Start, stop, and restart the vLLM inference server directly from the UI. Configure GPU memory utilization, quantization, tensor parallelism, and all vLLM CLI flags. Live GPU stats via nvidia-smi polling.

💬

Streaming Chat with Markdown

Token-by-token streaming output with blinking cursor. Full markdown rendering: tables, syntax-highlighted code blocks with copy buttons, math, and rich formatting. Auto-scroll to bottom.

💾

Model Discovery & Metadata

Automatically scan directories for HuggingFace-format models (safetensors) and GGUF files. Detects architecture, quantization (AWQ/GPTQ/FP8/GGUF), context length (RoPE scaling), vision support, and model size. Cached for instant retrieval.

📎

Image & PDF Attachments

Attach images (JPG, PNG, WebP, GIF) for vision-capable models via base64 image_url. Attach PDFs with automatic server-side text extraction. Enforced size limits and format validation.

🔍

Chat Organization

Folders with colour coding, session pinning, fuzzy search across titles and content, inline rename, drag-and-drop between folders, and bulk export/import as JSON.

🌎

OpenAI-Compatible API

Any application using the OpenAI SDK can point to VLLM Studio's API endpoint. Full /v1/chat/completions support including streaming, parameters, and stop sequences. Copy base URL with one click.

🤖

HuggingFace Hub Browser

Search the HuggingFace Hub for models directly within VLLM Studio. Browse quantized GGUF variants, download models with progress tracking, and filter by vLLM compatibility.

📈

VRAM Estimation Engine

Pre-flight resource planning tool that estimates weights, KV cache, activations, and overhead memory usage. Provides fit verdicts (full-gpu, partial-gpu, cpu-offload) and recommended context lengths.

🧰

GGUF Architecture Support

Pure TypeScript GGUF v3 binary metadata parser. One-click architecture patching for community models with non-standard labels. Auto-detects context length, embedding size, and block configuration from binary headers.

📖

Release Notes & About

Comprehensive release notes with search, section navigation, and roadmap. Full about page with author bio, tech stack, AI collaborator credits, and links to the vLLM documentation at docs.vllm.ai.

OpenAI-Compatible API

Once a model is loaded, VLLM Studio exposes an OpenAI-compatible chat completions endpoint. Any OpenAPI SDK-compatible application can use it directly.

# Python example using the openai package
from openai import OpenAI

client = OpenAI(
  base_url="http://localhost:8000/v1",
  api_key="not-needed"
)

response = client.chat.completions.create(
  model="your-model",
  messages=[{"role": "user", "content": "Hello!"}],
  temperature=0.7,
  max_tokens=512
)

print(response.choices[0].message.content)
# curl example
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model",
    "messages": [{"role": "user", "content": "Hello!"}],
    "temperature": 0.7,
    "max_tokens": 512,
    "stream": true
  }'

Security & Reliability

Local-Only Binding

Backend and vLLM server bind to 127.0.0.1 by default. External access requires explicit user opt-in via settings.

Path Sanitization

All model paths are validated against user-configured directories. Directory traversal attacks are blocked.

Input Validation

Zod schemas validate every API request body. Type-safe on both frontend (API client) and backend.

Graceful Error Handling

CUDA OOM errors are caught from vLLM stderr and surfaced as actionable user-facing messages. Auto-retry on crash (max 3x).

Crash Recovery

PID file-based re-attachment on backend restart. vLLM subprocess is detected and re-monitored if still running.

Database Reliability

SQLite WAL mode for concurrent reads. Automatic migrations with version tracking. No manual schema management.

Getting Started

# 1. Install system prerequisites (once)
sudo apt-get install -y nodejs npm python3 python3-pip python3-venv

# 2. Clone and launch — run.sh does everything:
# · Creates a Python venv/ (isolated from system)
# · Installs vLLM inside the venv
# · Installs all Node.js dependencies
# · Builds frontend and compiles backend
# · Launches the server
git clone <repo-url> && cd vllm-studio
bash run.sh

# 3. Open browser
http://localhost:3333

# 4. Add model directory in Settings
# 5. Select a model and start the vLLM server
# 6. Start chatting!

# Subsequent runs — run.sh detects the existing
# venv and skips re-installation. Just run:
bash run.sh
